# webapp-owasp-top10
A practical demonstration of OWASP Top 10 vulnerabilities with vulnerable and secure code implementations, including exploitation evidence and mitigation techniques.

Project Overview

This project demonstrates real world examples of vulnerabilities from the OWASP Top 10 list. It includes both vulnerable and secure code implementations to highlight how common web application security flaws occur and how they can be properly mitigated.

The repository contains:
- Vulnerable application code
- Secure implementation
- Exploitation evidence (screenshots)
- Vulnerability analysis report
- Remediation explanation

The tools used include:
- Kali Linux
- Burp Suite
- MariaDB
- apache2
- Foxyproxy


The Objectives of this project include to;

- Understand common web application vulnerabilities
- Demonstrate exploitation techniques in a controlled environment
- Compare insecure vs secure coding practices
- Provide practical remediation guidance


Covered OWASP Categories

- SQL Injection
- Broken Access Control
- Security Misconfiguration
- SSRF
- Insecure file upload

This project was created strictly for educational and research purposes. Do not deploy vulnerable code in production environments.
